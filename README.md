# Professional Data Entry & Data Cleaning Portfolio

Welcome to my professional Data Entry and Data Cleaning repository. This project serves as a practical demonstration of my core competencies in managing high-volume alphanumeric entry, identifying administrative data anomalies, executing data scrubbing procedures, and producing analytical-ready corporate spreadsheets.

## 📄 Project Overview
This project simulates a real-world administrative workflow by processing an e-commerce sales dataset consisting of **50 rows and 10 multi-format columns**. The workbook contains two comparative lifecycle stages:

1. **Sheet 1 (Data Before):** A raw, unvetted dataset filled with common human-input errors such as erratic date structures, nested/trailing whitespaces, non-standardized categorical terms, mixed text casings, and unautomated total valuations.
2. **Sheet 2 (Data After):** A finalized, corporate-standard dataset that has been completely audited, standardized, visually optimized, and dynamically calculated using native spreadsheet formulas.

---

## 🛠️ Core Competencies & Techniques Applied
To elevate the raw dataset into an elite business-ready standard, I systematically implemented the following data processing methodologies:

* **Advanced Data Trimming:** Executed thorough whitespace management to programmatically eliminate redundant leading, trailing, and double nested spaces (`TRIM`) across customer identifiers and contact points.
* **Categorical Standardization:** Audited and resolved inconsistent nomenclature by mapping varied text fragments (e.g., merging shorthand abbreviations like "elec", "ELECTRONICS", and "electronic") into uniform corporate categories using string standardization.
* **Text Case Regularization:** Normalization of mixed customer naming entries into a consistent corporate standard (`PROPER` case execution) to ensure CRM-ready synchronization.
* **Temporal Uniformity:** Audited chaotic, multi-regional date formats (alphanumeric, string, short date variations) and converted them into a unified international standard format (`YYYY-MM-DD`).
* **Automated Mathematical Computations:** Mitigated human accounting errors by shifting from hardcoded inputs to dynamic formulas (`Quantity * Unit Price`) to compute transactional line item valuations, combined with robust `=SUM()` functions for bottom-line financial reporting.
* **Aesthetic & Visual Optimization:** Designed the final output adhering to corporate presentation standards, featuring:
  * *Zebra Striping* (subtle alternating fills) to enhance scannability.
  * *Strict Accounting Alignments* (numeric values right-aligned, strings left-aligned, transactional IDs and dates centered).
  * Professional currency mapping (`$#,##0.00`) and quantity formatting.
  * *Freeze Panes* implementation for seamless high-volume row navigation.
